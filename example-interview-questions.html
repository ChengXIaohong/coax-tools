第 1 题：请解释一下操作系统中的进程和线程的区别？

🎯 考察点

  • 进程是资源分配的最小单位
  • 线程是 CPU 调度的最小单位
  • 线程共享进程资源
  • 进程间相互独立

💡 参考答案

进程和线程的主要区别在于： 1. 进程是操作系统资源分配的基本单位，而线程是 CPU 调度的基本单位 2. 一个进程可以包含多个线程，这些线程共享进程的内存空间和资源 3. 进程之间有独立的地址空间，一个进程崩溃不会影响其他进程；而同一进程的线程共享内存，一个线程崩溃可能导致整个进程崩溃 4. 进程切换开销较大，线程切换开销较小 5. 进程间通信需要特殊的 IPC 机制，线程间可以直接通过共享变量通信

🔍 拓展范围

进程同步与互斥 死锁问题 线程池 协程概念

第 2 题：什么是虚拟内存？它有什么作用？

🎯 考察点

  • 抽象的内存概念
  • 扩展可用内存
  • 内存保护
  • 简化编程模型

💡 参考答案

虚拟内存是计算机系统内存管理的一种技术，它使得应用程序认为它拥有连续可用的内存（一个连续完整的地址空间），而实际上物理内存通常是被分割成多个碎片存储在磁盘上的。 主要作用： 1. 扩展内存：允许运行比物理内存更大的程序 2. 内存隔离：每个进程有独立的虚拟地址空间，提高安全性 3. 简化编程：程序员无需关心物理内存的分配和管理 4. 提高内存利用率：通过分页机制，只加载需要的部分到内存

🔍 拓展范围

页表机制 TLB 快表 页面置换算法 缺页中断

第 3 题：解释一下缓存的工作原理以及常见的缓存替换算法

🎯 考察点

  • 局部性原理
  • 多级缓存
  • 命中率
  • 替换策略

💡 参考答案

缓存工作原理： 基于程序的局部性原理（时间局部性和空间局部性），将频繁访问的数据存储在更快但容量较小的存储器中。 常见替换算法： 1. FIFO（先进先出）：淘汰最早进入缓存的数据 2. LRU（最近最少使用）：淘汰最长时间未被访问的数据 3. LFU（最不经常使用）：淘汰访问频率最低的数据 4. OPT（最优替换）：淘汰未来最长时间内不会被访问的数据（理论算法） 5. Clock 算法：LRU 的近似实现，性能较好

🔍 拓展范围

缓存一致性 缓存穿透/击穿/雪崩 CPU 缓存行 分布式缓存

第 4 题：什么是数据库的事务？事务的 ACID 特性是什么？

🎯 考察点

  • 原子性
  • 一致性
  • 隔离性
  • 持久性

💡 参考答案

事务是数据库管理系统执行过程中的一个逻辑单位，由一个有限的数据库操作序列构成。 ACID 特性： 1. 原子性（Atomicity）：事务是一个不可分割的工作单位，要么全部执行，要么全部不执行 2. 一致性（Consistency）：事务必须使数据库从一个一致性状态变换到另一个一致性状态 3. 隔离性（Isolation）：一个事务的执行不能被其他事务干扰 4. 持久性（Durability）：一旦事务提交，其对数据库的修改就是永久性的，即使发生故障也不会丢失

🔍 拓展范围

事务隔离级别 并发异常 MVCC 锁机制

第 5 题：解释一下计算机网络中的 OSI 七层模型

🎯 考察点

  • 分层架构
  • 各层功能
  • 协议封装
  • 数据传递

💡 参考答案

OSI 七层模型从下到上分别是： 1. 物理层：传输原始比特流，定义电气、机械、规程和功能特性 2. 数据链路层：建立逻辑连接，帧的传输和差错控制（如以太网） 3. 网络层：路径选择和寻址（如 IP 协议） 4. 传输层：端到端的可靠或不可靠传输（如 TCP、UDP） 5. 会话层：建立、管理和终止会话 6. 表示层：数据格式转换、加密解密、压缩解压缩 7. 应用层：为应用程序提供网络服务（如 HTTP、FTP、SMTP） 每层利用下一层的服务，向上一层提供接口
